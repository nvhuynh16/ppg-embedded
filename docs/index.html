What's in this repo

  • Three Cortex-M3 firmware entry points — peak detector (main.c), FFT spectral path with sub-harmonic check (main_fft.c), BW-path respiration estimator (main_rr.c) — all sharing one Q15 DSP layer.
  • FFT sub-harmonic check — catches PPG morphologies where the 2nd harmonic carries more energy than the fundamental (otherwise the FFT path locks at 2× HR). Without the check, MAE 2.43 bpm; with it, 1.72 bpm.
  • Q15 dynamic-range proof — closed-form bounds for every Q-format variable + Monte-Carlo validator. docs/qformat_proof.md
  • Unity unit tests + gcov — 19 tests, 100 % line coverage on dsp_fixed.c + dsp_fft.c.
  • Patient-cluster bootstrap CIs — every published metric is a range, not a point. See src/bootstrap.py for why cluster vs window resampling.
  • Bit-exactness contract — the Q15 FFT in firmware/dsp_fft.c matches numpy.fft.fft within log₂(N)+1 LSB per bin. The Python golden reference and the C firmware share the same algorithmic invariants.
